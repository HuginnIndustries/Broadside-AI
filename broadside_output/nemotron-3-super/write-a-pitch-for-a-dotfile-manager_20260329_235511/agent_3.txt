### ✅ What DotSync Does Differently

🔹 **Declarative, Not Imperative**  
Define your desired state in a simple `dotsync.yaml`—no more guessing which script ran last or why your prompt disappeared.  
```yaml
# dotsync.yaml
files:
  - source: ~/.zshrc
    target: ~/dotfiles/zshrc
    template: true  # enables variable substitution
  - source: ~/.config/nvim/init.lua
    target: ~/dotfiles/nvim/init.lua
secrets:
  - source: ~/.ssh/id_ed25519
    encrypted: true  # auto-encrypted with age/GPG, decrypted on sync
hooks:
  post-sync:
    - nvim --headless -c 'PlugInstall' -c 'qa'
```

🔹 **Smart Templating & Secrets Management**  
Use Jinja2-like syntax to conditionally include configs based on OS, hostname, or role (work/laptop/server). Secrets are auto-encrypted at rest and decrypted *only* during sync—never committed plaintext.

🔹 **Cross-Platform, Zero Dependencies**  
Written in Rust for speed and safety. No Python, no Node, no Ruby. Just a single binary: `dotsync sync`, `dotsync backup`, `dotsync restore`.

🔹 **Integrated with Your Workflow**  
Hook into your shell:  
```bash
# Add to .zshrc
precmd() { dotsync check && echo "✅ Dotfiles in sync" || echo "⚠️  Dotfiles drift detected!" }
```
Get instant feedback when your config drifts—no more surprise breakages.

🔹 **Backup & Rollback Made Safe**  
Every sync creates an immutable, encrypted snapshot. Roll back to any previous state with `dotsync rollback --to 2024-06-15`.

🔹 **Community & Extensible**  
Official templates for popular tools (Neovim, Wezstar, Starship, Fish, Hyprland). Write your own plugins in Rust or WASM to extend behavior—like auto-installing fonts or applying GSettings.
