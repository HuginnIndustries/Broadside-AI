### ✅ What DotManager Does (So You Don’t Have To)
- **One Command Setup**: `dotmanager init` scans your home directory, intelligently detects common dotfiles, and proposes a clean, version-controlled structure — no guesswork.
- **Declarative, Not Imperative**: Define your desired state in a simple `dotmanager.yml`:
  ```yaml
  files:
    - src: ~/.bashrc
      dest: configs/shell/bashrc
    - src: ~/.gitconfig
      dest: configs/git/gitconfig
    - src: ~/.config/nvim
      dest: configs/neovim
  ```
  DotManager handles symlinking, backups, and conflict resolution — automatically.
- **Cross-Platform & Shell-Agnostic**: Works on macOS, Linux, and WSL. No Bash/Zsh/Fish bias — it just works.
- **Smart Conflict Detection**: Before overwriting, DotManager shows you a diff, backs up the original, and lets you approve, edit, or skip — no silent data loss.
- **Encrypted Secrets (Optional)**: Store API keys, SSH configs, or AWS credentials in a `.dotmanager/secrets/` vault — encrypted with age or GPG, decrypted only on your trusted machines.
- **Sync Across Devices**: Push your dotfiles to any Git host (GitHub, GitLab, self-hosted). `dotmanager sync` pulls and applies changes securely — ideal for devs jumping between work, home, and cloud machines.
- **Plugin Ecosystem**: Want to auto-install Homebrew brews, npm globals, or VS Code extensions? DotManager’s plugin system lets you extend behavior without forking the core.
