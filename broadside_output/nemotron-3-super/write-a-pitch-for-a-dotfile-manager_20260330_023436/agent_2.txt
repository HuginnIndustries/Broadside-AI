### 🎯 The Problem  
Dotfiles are the soul of your developer environment—your shell, editor, aliases, and workflows. But managing them is a minefield:  
- ❌ Manual syncing via USB or cloud drives = version drift  
- ❌ Git repos with hardcoded paths = fragile and non-portable  
- ❌ Over-engineered tools = steep learning curves, bloated dependencies  
- ❌ Risk of leaking secrets (API keys, tokens) in public repos  

You shouldn’t have to choose between convenience and security. You shouldn’t waste hours rebuilding your setup after a clean install. You deserve a system that *just works*.

---

### ✨ The Solution: DotWeaver  
*DotWeaver* is a lightweight, zero-config, privacy-first dotfile manager that turns your configuration into a portable, version-controlled, and securely synchronized experience—without the complexity.

#### 🔑 Core Features:  
- **Smart Symlinking, Not Copying**  
  DotWeaver intelligently symlinks your dotfiles from a central `~/.dotweaver` repo to their expected locations—preserving original file permissions, ownership, and editor integration. No more “why isn’t my alias working?” headaches.

- **Secrets? Encrypted & Local-First**  
  Sensitive data (like SSH keys, AWS credentials, or API tokens) stays *encrypted at rest* using your GPG key or age encryption—never uploaded unless you explicitly opt-in. Your secrets are yours alone.

- **Git-Backed, But Not Git-Dependent**  
  Your dotfiles live in a standard Git repo (GitHub, GitLab, self-hosted)—so you get full history, branching, and rollback. But DotWeaver abstracts the Git layer: you don’t need to know Git internals to use it. `dotweaver sync` does the heavy lifting.

- **Cross-Platform & Shell-Agnostic**  
  Works on macOS, Linux, and WSL2. Bash, Zsh, Fish, PowerShell—DotWeaver adapts. No need to rewrite configs per shell.

- **Profiles & Environments**  
  Define *work*, *personal*, *server*, or *testing* profiles. Toggle between them with `dotweaver use work`—perfect for contractors, consultants, or multi-machine workflows.

- **Pre/Post Hooks & Validation**  
  Run scripts before/after sync (e.g., install plugins, reload shell, run `brew bundle`). Built-in linting warns you if a file looks broken before linking.

- **Zero Dependencies, Single Binary**  
  Written in Rust for speed and safety. One 8MB binary. No Node, no Python, no Docker. Install via `brew install dotweaver` or download the release—done.

### 🚀 Get Started in 60 Seconds  
```bash
# Install
brew install dotweaver   # or: curl -LSfs https://dotweaver.dev/install.sh | sh

# Initialize
dotweaver init           # creates ~/.dotweaver/repo and walks you through setup

# Add your first file
dotweaver add ~/.zshrc   # symlinks it, adds to repo, commits

# Sync to any machine
dotweaver clone git@github.com:you/dotfiles.git
dotweaver sync           # pulls, decrypts secrets (if you allow), symlinks
```

No account needed. No telemetry. No vendor lock-in. Just your config, your way—everywhere.
