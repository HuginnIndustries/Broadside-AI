### **The Problem**

Every developer, sys‑admin, or power‑user knows the pain:

* **Scattered configs** – your `.vimrc`, `.tmux.conf`, `zshrc`, VS Code settings, and dozens of hidden files live in random folders on every machine.

* **Security concerns** – API keys, tokens, and private snippets get baked into the repo, exposing you when you push to GitHub or share with a teammate.
* **No single source of truth** – each machine drifts, leading to “works on my machine” frustration.

These pains cost **hours of onboarding, countless lost tweaks, and security headaches**—and they’re growing as your toolchain expands.

### **Demo (60‑Second Walkthrough)**

1. **Clone**: `curl -fsSL https://dotforge.dev/install | bash`
2. **Initialize**: `dotforge init` – answers a few prompts (OS, shell, editors) and creates `dotforge.yml`.
3. **Add a Config**: `dotforge add vimrc --src ~/myconfigs/vimrc` – automatically creates a symlink and adds it to Git.
4. **Secure a Secret**: `dotforge secret add github_token` – stores it encrypted in `secrets.yml`.
5. **Deploy**: `dotforge apply` – installs required packages, decrypts secrets, links files, and prints a beautiful summary.
6. **Rollback**: `dotforge revert HEAD~1` – restore previous state instantly.

All of this happens **offline** (except optional secret sync), and every step is logged in the built‑in Git history.
