## **DotKeeper – Your Personal Configuration Concierge**  

### **One‑Liner**
*“Never lose a setting again. DotKeeper turns your dotfiles into a living, breathing, version‑controlled companion that follows you from laptop to server, from Mac to Linux to WSL.”*  

---

### **Why It Matters**
- **Configuration chaos is real.** Every new machine means copy‑pasting, hunting for a forgotten `~/.bashrc`, or worse—breaking a production environment.
- **Time is money.** The average dev spends **2–3 hours a month** manually syncing configs. That’s over **20 hours a year** wasted on repetitive copy‑pastes.
- **Security & reproducibility** are non‑negotiable in today’s CI/CD pipelines. A missing alias or a stray SSH key can break builds or expose credentials.

**DotKeeper eliminates all of that.**

### **The Core Value Proposition**

| Problem | DotKeeper Solution | Impact |
|---------|-------------------|--------|
| **Scattered files** across `$HOME`, `/etc`, and app‑specific directories | **Unified repository** with a smart manifest that maps any file, folder, or symlink to a logical name | One command `dk sync` restores *everything* instantly |
| **Manual copy‑paste** when setting up a new machine | **One‑click bootstrap** that pulls the latest snapshot from your private remote (Git, Gitea, or self‑hosted) | New machine ready in **≤30 seconds** |

| **Cross‑platform quirks** (macOS vs Linux vs WSL) | **Conditional blocks & templating** (Jinja2‑style) that render the correct file per OS, distro, or hostname | One source of truth, zero duplicate files |
| **Hard to onboard teammates** | **Team mode** – shared repo with fine‑grained permissions, audit logs, and a CLI wizard that validates env before applying | Faster onboarding, fewer “my dotfiles break the build” tickets |

---

### **Key Features in a Nutshell**

1. **Zero‑Config Start** – `dk init` detects existing dotfiles, suggests a manifest, and creates a ready‑to‑push Git repo in seconds.  
2. **Atomic Sync Engine** – Handles symlinks, directories, and executable flags, guaranteeing a **transactional** apply: either everything succeeds or nothing changes.  
3. **Smart Templates** – Use `{{ os }}`, `{{ hostname }}`, `{{ env.VAR }}` inside any file; DotKeeper renders a perfect per‑host version on the fly.  
4. **Secret Management** – `dk secret add <key>` encrypts with *age* (or PGP) and stores the cipher text alongside your dotfiles; decryption occurs only on the target host.  
5. **Multi‑Backend Support** – GitHub, GitLab, Gitea, Bitbucket, or any **bare SSH repo**; plus optional **S3/Backblaze** fallback for large binary assets (e.g., custom scripts).  
6. **Rich CLI + TUI** – `dk status`, `dk diff`, `dk log`, and a curses‑based UI for visualizing the manifest, pending changes, and secrets.  
7. **Extensible Plugins** – Write a plugin in any language; ship as a binary or a simple shell script. Official plugins already cover: `zsh`, `tmux`, `alacritty`, `gitconfig`, `vscode`; the ecosystem is growing fast.  
8. **Continuous‑Delivery Friendly** – `dk apply --ci` runs headless, exits non‑zero on drift, and can be used in CI pipelines to validate that production boxes match the repo.
