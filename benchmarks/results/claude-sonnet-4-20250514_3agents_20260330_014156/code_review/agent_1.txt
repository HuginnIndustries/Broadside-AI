## **CRITICAL Issues**

### 1. SQL Injection Vulnerability
**Severity: CRITICAL**
```python
query += f" AND {key} = '{val}'"
```
- User input is directly interpolated into SQL query without sanitization
- Attackers can execute arbitrary SQL commands
- **Fix**: Use parameterized queries or ORM with proper escaping

### 2. Mutable Default Argument
**Severity: CRITICAL**
```python
def get_users(db, filters={}):
```
- Shared mutable object across function calls leads to persistent state
- Subsequent calls will retain filters from previous calls
- **Fix**: Use `filters=None` and check inside function
